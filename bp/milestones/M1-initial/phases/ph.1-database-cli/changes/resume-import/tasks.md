# Tasks: resume-import

> This document breaks design into executable tasks grouped by wave. Each task references design items (DS-N), spec_ref, files, acceptance. Every `type:behavior` task ships full RED→GREEN→REFACTOR protocol.

## TDD Type Annotations

| type          | Meaning                                                                  | TDD Protocol                                                                                            |
| ------------- | ------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------- |
| `behavior`    | Business behavior — implement concrete, observable/assertable feature   | **RED→GREEN→REFACTOR** (mandatory: test first → implement → refactor)                                  |
| `config`      | Configuration — env vars, CI/CD, lint, tsconfig, etc.                    | Direct implementation, no TDD                                                                          |
| `refactor`    | Refactoring — improve internal structure without changing behavior       | Verify tests pass → refactor → verify again                                                            |
| `docs`        | Documentation — README, API docs, comments                                | Direct implementation, no TDD                                                                          |
| `scaffolding` | Skeleton code — new module shells, directory structure, templates        | Direct implementation, no TDD                                                                          |

> **Rule**: if a task's core output is "a behavior" (user-perceptible,
> test-assertable), use `behavior`. If it's just "file exists" or "config
> takes effect", use `config`/`scaffolding`.

- [ ] T-5: [type:behavior] ResumeService.importFromFile rejects invalid paths, extensions, and content
  - **refs**: DS-1
  - **files**: `src/services/resume-service.ts`, `src/services/resume-service.test.ts`
  - **spec_ref**: specs/resume/spec.md SHALL-1
  - **acceptance**:
    - Empty string path → `MiValidationError` matching `/路径不能为空/`
    - Non-existent path → `MiValidationError` matching `/文件不存在/`
    - Existing directory (not file) → `MiValidationError` matching
      `/不是文件/`
    - Unsupported extension (`.txt`, `.docx`, no extension) →
      `MiValidationError` matching `/不支持的文件类型/`
    - File exists but is empty (0 bytes) →
      `MiValidationError` matching `/文件内容为空/`
    - PDF that fails to parse → `MiValidationError` matching
      `/PDF 解析失败/` (exit 1; user can supply a different file)
    - All error cases leave `profiles` row and `resume_history`
      table unchanged
  - **RED test**:
    ```text
    GIVEN profile id='P1' with resume_text='', resume_path=NULL
    WHEN service.importFromFile('')                THEN MiValidationError(/路径不能为空/)
    WHEN service.importFromFile('/no/such/file')   THEN MiValidationError(/文件不存在/)
    WHEN service.importFromFile('/tmp')            THEN MiValidationError(/不是文件/)
    WHEN service.importFromFile(<fixture.txt>)     THEN MiValidationError(/不支持的文件类型/)
    WHEN service.importFromFile(<empty.md>)        THEN MiValidationError(/文件内容为空/)
    WHEN service.importFromFile(<broken.pdf>)      THEN MiValidationError(/PDF 解析失败/)
    AND after every call: SELECT resume_text FROM profiles WHERE id='P1' === ''
    AND after every call: SELECT count(*) FROM resume_history === 0
    ```

  - **refs**: DS-2
  - **files**: `src/commands/resume.ts`, `src/commands/resume.test.ts`
  - **spec_ref**: specs/resume/spec.md SHALL-4
  - **acceptance**:

    - `runResumeCommand(['import', '--file', '/tmp/r.md'])` calls

    - Service throws `MiValidationError` → exits 1 with Chinese error
    - Service throws `MiDatabaseError` → exits 2 with `系统错误: `

  - **RED test**:
    ```text

    AND  stdout contains '已导入简历'
    AND  exit code 0

    GIVEN injected service.importFromFile rejects MiValidationError('PDF 解析失败: bad')
    WHEN runResumeCommand(['import', '--file', '/tmp/r.pdf'])
    THEN exits 1, stderr contains 'PDF 解析失败'

    WHEN runResumeCommand(['import'])
    THEN throws MiValidationError(/用法错误/)

    WHEN runResumeCommand(['import', '--file', '/x', '--profile', 'PID'])
    THEN service.importFromFile called with '/x', { profileId: 'PID' }
    ```
